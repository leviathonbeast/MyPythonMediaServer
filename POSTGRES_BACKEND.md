## Goal recap

Two pieces of work landed together on this branch:

1. **Optional Postgres backend** alongside SQLite, chosen by URL scheme
   (`sqlite://` or `postgresql://`). User wanted to keep both behind a
   config flag.
2. **MusicBrainz ID support** end-to-end — scan/store/serve.

Plus a bonus: the migration sequence was **squashed** into a fresh
`schema.sql` baseline because the user is happy with a fresh install
and 17 micro-migrations were paying no rent.

## Notes from this session

- We confirmed psycopg3 doesn't accept `:name` natively — that drove
  the connection wrapper's regex translation approach. Don't switch to
  positional `%s` thinking it's "more native"; the named form keeps
  queries.py readable and the regex is tiny.
- The `INTEGER PRIMARY KEY AUTOINCREMENT` vs `IDENTITY` difference is
  why we need separate schema files rather than a single portable one.
  SQLite has special rowid semantics that don't map cleanly.
- FTS5 → tsvector is the biggest single piece of remaining work. Plan
  on ~3-4 evenings for that alone.
